Artefakt

VibeSwipe Travel

Eine KI-gestützte Reise-App nach dem Tinder-Prinzip: Statt endloser Suchfilter swipet der User atmosphärische Bilder — die KI analysiert die visuellen Vorlieben und empfiehlt drei perfekt passende Reiseziele. Kein Suchfeld. Kein Stress. Nur Vibes.

VibeSwipe · Live auf Vercel vibeswipe-travel-app.vercel.app ↗

Iteration

VISION.md als Context Baseline

Vor dem ersten Prompt wurde ein vollständiges VISION.md Manifest erstellt — Zielgruppe, Kernfunktionen, Architektur und MVP-Phasen. Das war der entscheidende Unterschied zu blindem Vibe Coding.

Context Baseline

VISION.md als erster Prompt mitgegeben — die KI hatte sofort Kontext, weniger Halluzinationen, bessere Outputs von Anfang an.

Zoom-In Methode

Iterativ: (1) Makro-Layout, (2) UI-Komponenten verfeinern, (3) Swipe-Animationen und Haptik. Nie alles auf einmal prompten.

Pexels API Integration

Echte Reisebilder statt statischer Platzhalter. API-Key in .env gesetzt — Bolt hat das Setup automatisch gehandhabt.

Plan Mode & Steering

Komplexe Features zuerst als Text-Plan. Error-Logs direkt in den Chat kopiert — effizienter als blind weiterzupromten.

Reflexion

Learnings

Die Vision zählt: Das VISION.md Manifest war mehr als Dokumentation — es war der Schlüssel zu konsistenten KI-Outputs. Ohne Kontext ist Vibe Coding frustrierend.

Bolt → Vercel: Bolt eignet sich perfekt für schnelle Prototypen, Vercel für die finale Deployment-Umgebung. Beide Tools zusammen ergeben einen effizienten Workflow.

APIs sind einfacher als gedacht: Pexels API war in wenigen Minuten eingebunden — die grösste Hürde war das Verstehen was die API überhaupt macht, nicht die technische Integration.

Gemini-Tagging in Echtzeit war zu langsam für flüssiges Swipen — die KI-Analyse läuft deshalb gebündelt nach dem Swipen, nicht pro Karte.

Zero-Search UX funktioniert: Das "kein Suchfeld"-Prinzip ist intuitiver als erwartet — Nutzer verstehen die Swipe-Mechanik sofort ohne Erklärung.

Bolt.new als Gamechanger: Kein lokales Setup, kein Terminal-Stress — Bolt ermöglichte es, innerhalb von Minuten einen funktionierenden Prototyp mit Datenbank und Edge Functions zu haben. Ideal für schnelles Experimentieren ohne technische Hürden.
